Профессиональный, полностью асинхронный Telegram-бот для управления экосистемой сервиса доставки еды формата "Dark Kitchen". Архитектура построена с нуля на основе принципов **Domain-Driven Design (DDD)** с использованием высокопроизводительного асинхронного Python.

## 🎯 Цель проекта
Предоставить комплексное алгоритмическое и инфраструктурное решение для службы доставки полного цикла. Бот управляет точным потоком заказов и выступает единой точкой контакта для четырех ключевых ролей: **Клиентов, Кассиров, Поваров (Шефов) и Курьеров**.

## 🏗 Архитектура
Проект строго придерживается многослойной архитектуры Clean/Domain-Driven Design (DDD):
- **`domain`**: Базовые сущности, объекты-значения и бизнес-правила (например, доменные модели SQLAlchemy).
- **`infrastructure`**: Абстрактные репозитории (слой доступа к данным), сессии кэширования в Redis, интеграции с внешними API и настройки базы данных.
- **`application`**: Сценарии использования (Use Cases) и фоновые процессы (Celery: автоназначение курьеров, расчет умного ETA).
- **`presentation`**: Telegram-хэндлеры Aiogram, контексты конечных автоматов (FSM) и Middleware (внедрение зависимостей, проверка ролей).

## ✨ Ключевые возможности
- **Умный ролевой доступ (RBAC)**: Middleware мгновенно маршрутизирует пользователей в зависимости от их назначенных ролей (Client, Cashier, Chef, Courier, Admin, SuperAdmin).
- **Асинхронная корзина покупок**: Сверхбыстрое временное хранилище корзины реализовано исключительно в оперативной памяти с использованием кластеров Redis.
- **Автоматизированный процесс через Celery**:
  - Динамический расчет ETA (времени доставки) на основе текущей загруженности кухни.
  - Автоматическое распределение заказов между Курьерами, привязанными к конкретным зонам доставки.
  - Активация предзаказов по расписанию (CRON).
- **Внедрение зависимостей (DI Pipeline)**: Сессии БД и пулы Redis централизованно управляются и "чисто" внедряются в хэндлеры Aiogram, исключая утечки соединений.
- **Нативная интеграция оплат**: Полная поддержка Telegram Payments UI для быстрых и безопасных безналичных платежей.

## 🚀 Быстрый старт

### 1. Требования
- Python 3.10+
- Docker и Docker Compose
- Poetry (Менеджер пакетов)

### 2. Установка
```bash
# Клонировать репозиторий
git clone https://github.com/ваш-юзернейм/neopizza-dark-kitchen.git
cd neopizza-dark-kitchen

# Инициализировать виртуальное окружение и установить зависимости
poetry install

# Настроить локальные переменные окружения
cp .env.example .env
# Отредактировать .env, добавив ваш BOT_TOKEN и Telegram ID
```

### 3. Запуск сервисов
Для запуска локальных контейнеров PostgreSQL и Redis выполните:
```bash
docker-compose up -d
```

### 4. Миграции Базы Данных
Примените первоначальную схему базы данных:
```bash
poetry run alembic upgrade head
```

### 5. Запуск бота
Запустите основное приложение:
```bash
poetry run python -m src.presentation.bot
```

## 📈 Масштабируемость
- **Stateless Хэндлеры**: Экземпляры бота не хранят состояние внутри себя. Вся информация сохраняется во внешнем Redis, что позволяет легко масштабироваться горизонтально на несколько серверов.
- **Пул соединений с БД**: Встроенный асинхронный пул соединений SQLAlchemy (`asyncpg`).
- **Распределенные очереди задач**: Тяжелые операции (уведомления, гео-маршрутизация, формирование инвойсов) вынесены в Celery-воркеры, которые масштабируются независимо от API бота.
- **Безопасность Production-уровня**: Все эндпоинты защищены шлюзами приватности (Privacy Gateways). Токены и пароли строго вынесены в `.env`. Hệ thống поддерживает строгую статическую типизацию (`mypy`).
