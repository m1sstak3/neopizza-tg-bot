# Portfolio Specification: Telegram-бот доставки еды "NeoPizza Dark Kitchen"

## 1. Executive Summary
Продукт представляет собой D2C (Direct-to-Consumer) платформу доставки еды на базе Telegram. Формат — **100% Dark Kitchen с опцией самовывоза**. Бот берет на себя роль единого интерфейса для клиентов и легкой ERP-системы для персонала (интеграция с локальной кассой ручная, бухгалтер получает отдельный поток данных по ингредиентам и выручке). Основная бизнес-цель: уйти от 20-35% комиссии агрегаторов, создав ультимативно быстрый и бесшовный UX заказа пиццы (чекаут за 3 клика), а также снизить планку минимального чека за счет самовывоза. Для защиты данных реализуется экран принятия Privacy Policy (ФЗ-152 / GDPR) при первом запуске.

## 2. Business Direction Analysis
Среди рассмотренных вариантов (шаурмичная, столовая, кафе, ресторан, пиццерия) выбрана **Сетевая Пиццерия**. 
* **Обоснование:** Пицца идеально выдерживает транспортировку, обладает высокой маржинальностью (Food Cost ~20-25%) и легко преодолевает минимальный порог заказа (1200 ₽) даже на одного-двоих человек. Внедрение самовывоза полностью снимает блокер минимального заказа для одиночных клиентов, повышая общую конверсию. Операционная модель пиццерии (конвейер) линейно ложится в спроектированную FSM-архитектуру телеграм-бота.

## 3. Clarifying Questions (Дискавери-интервью завершено)
По результатам проведенного интервью закрыты ключевые неопределенности:
* Отказ от залов посадки (только доставка/самовывоз).
* Динамическое ценообразование логистики по геозонам, никаких сторонних доставок.
* Обработка пиковых нагрузок через формулу динамического ETA.
* Выделение отдельной информационной роли "Бухгалтер".
* Защита от курьерского фрода (блокировка только через ручной review).

## 4. Selected Concept & Positioning
**Название:** NeoPizza (рабочее).
**Формат:** Dark Kitchen + Самовывоз.
**Позиционирование:** Супер-быстрая технологичная пицца без приложений и звонков. Идеально для digital-native аудитории, геймеров, айтишников, офисных работников. Минималистичный текстово-визуальный интерфейс бота, акцент на скорость и предсказуемость. Заказ делается без отрыва от мессенджера.

## 5. Full Roadmap
### Phase 1 — Market, Concept & Legal (Основа)
* Согласование Privacy Policy экрана (Сбор ПД).
* Фиксация геозон и тарифов (Радиус/Цена).
* Формирование алгоритма динамического ETA (Заказы * Время_приготовления + Логистика).

### Phase 2 — Architecture & Database Design (Технический фундамент)
* Проектирование БД (PostgreSQL) с учетом новой сущности: Данные для Бухгалтерии (номенклатуры, расчет костов).
* Развертывание CI/CD и Docker-инфраструктуры для GitHub портфолио.

### Phase 3 — Core Telegram Bot & Client Flow (Клиент)
* Экран "Согласие на обработку ПД".
* Выбор режима: Доставка по зонам (с проверкой попадания в полигон) / Самовывоз.
* Каталог, корзина (Redis), чекаут с ограничением 1200₽ (для доставки). 
* Алерт о высокой нагрузке при пиковых ETA с предложением самовывоза.

### Phase 4 — Staff Operations & Logistics (Персонал)
* Интерфейсы Шефа и Кассира. 
* Разработка интерфейса для Бухгалтера (выгрузка детальных чеков/ингредиентов).
* Курьерское приложение в боте: биржа заказов, статусы.
* Механика отправки "жалоб курьера" в чат Админа для ручного ревью.

### Phase 5 — Admin Panel & Launch (Запуск)
* Разработка админки для управления меню, персоналом и блокировками (разбор инцидентов).
* Интеграция онлайн-оплат (Telegram Payments API).
* Тестирование нагрузок и релиз.

## 6. Financial Model Structure (Юнит-экономика)
**Ключевые показатели на 1 заказ (AOV ~ 1500 ₽):**
* **Доходы:** Онлайн и оффлайн оплата. Доля самовывоза (ожидаемо 25%) увеличит маржинальность заказа, так как логистический кост = 0.
* **COGS (Продукты):** ~300-350 ₽ (20-25%).
* **Эквайринг:** ~2.5% (37 ₽).
* **Логистика доставки:** Оплата курьеру зависит от геозоны (в среднем ~250 ₽).
* **Contribution Margin (на доставке):** ~55-60%.
* **Contribution Margin (на самовывозе):** ~75%.
* **Фиксированные косты (OPEX):** Аренда Dark Kitchen, ФОТ (Сотрудники, Бухгалтер).
* Модель устойчива к пиковым нагрузкам за счет системы алертов: мы не теряем лояльность (и деньги на возвратах) при перегрузах, честно предупреждая клиента.

## 8. Risk Register
| Риск | Влияние | Стратегия смягчения (Mitigation) |
| :--- | :--- | :--- |
| **Пиковая нагрузка** | High | Внедрен алгоритм расчета ETA. Обязательный алерт пользователю "Время ожидания N. Согласны или выберете самовывоз?". |
| **Фрод курьера (ложный невыкуп)** | Medium | Все запросы курьеров на блокировку падают Админу на Manual Review. Никаких автоматических банов. |
| **Юридический (ПД)** | Critical | Запрет на использование бота до нажатия кнопки [Принимаю согласие на обработку ПД] в самом начале. |
| **Недостижение мин. чека** | Low | Введена функция самовывоза без лимитов. В случае доставки бот мягко допродает напитки до суммы 1200 ₽. |

## 9. Assumptions (Допущения до старта)
1. Кассир вручную переносит данные о заказе из Telegram в стационарную кассовую систему (Интеграция по API с кассой за рамками первой версии).
2. Расчет ингредиентов для Бухгалтера генерируется ботом на основе зашитых в БД технологических карт блюд.
3. Расчет геозоны доставки использует прямую математическую дистанцию (Haversine) или API матриц маршрутов (OSRM).
